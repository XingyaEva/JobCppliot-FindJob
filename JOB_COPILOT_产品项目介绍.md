# Job Copilot 智能求职助手 - 产品项目介绍

> **项目定位**：面向 AI/产品/技术类求职者的全链路智能求职助手  
> **产品版本**：v1.0.0（Phase 9 - 投递跟踪已完成）  
> **项目角色**：产品负责人 & 独立开发者  
> **上线地址**：https://job-copilot.pages.dev

## 一、项目概述

### 1.1 项目背景与痛点洞察

在互联网行业求职过程中，求职者面临以下核心痛点：

| 痛点 | 描述 | 影响 |
|------|------|------|
| **JD解析困难** | 岗位描述信息分散，难以快速提取核心要求 | 筛选效率低，错过优质机会 |
| **匹配度模糊** | 不清楚自身与岗位的匹配程度 | 海投无效，面试准备无方向 |
| **简历针对性差** | 一份简历投所有公司，缺乏定向优化 | 简历通过率低 |
| **面试准备盲目** | 不了解公司背景，面试问题准备无针对性 | 面试表现不佳 |
| **投递管理混乱** | 投递记录分散，状态跟踪困难 | 错过面试机会 |

### 1.2 产品愿景

**打造求职者的 AI 副驾驶**，覆盖从岗位发现到面试准备的全链路，通过 AI 技术提升求职效率和成功率。

### 1.3 目标用户画像

- **主要用户**：互联网行业求职者（产品经理、开发工程师、设计师等）
- **使用场景**：
  - 海投前的岗位筛选与匹配度评估
  - 面试前的公司调研与问题准备
  - 简历的针对性优化

### 2.2 核心功能模块详解

#### 模块一：JD 智能解析（多模态输入 + 双维度分析）

**功能描述**：支持截图OCR和文本输入两种方式，自动解析岗位描述，输出结构化分析结果。

**核心能力**：
- 📸 **多模态输入**：支持图片OCR识别 + 文本直接粘贴
- 🔗 **URL爬取**：支持 Boss直聘、拉勾、猎聘等主流招聘平台链接自动解析
- 🏗️ **结构化提取**：自动提取岗位名称、公司、薪资、职责、要求等关键信息

**A/B 双维度分析体系**（原创设计）：

| 维度 | 分析内容 | 分析价值 |
|------|----------|----------|
| **A1 技术栈** | 技术关键词提取 + 技术密度判断 | 判断岗位技术深度要求 |
| **A2 产品类型** | ToB/ToC/平台产品分类 | 快速定位产品形态 |
| **A3 业务领域** | 行业/领域识别 | 评估行业经验匹配度 |
| **A4 团队阶段** | 0-1/1-10/成熟期判断 | 判断能力要求侧重点 |
| **B1 行业背景** | 行业经验硬性/软性要求 | 评估核心匹配门槛 |
| **B2 技术要求** | 技术深度分级（了解/熟悉/精通） | 精准定位技术差距 |
| **B3 产品经验** | 全周期/0-1经验要求分析 | 评估经验匹配度 |
| **B4 能力要求** | 核心产品能力拆解 | 面试准备重点方向 |

#### 模块二：简历智能解析（MinerU + 能力标签）

**功能描述**：支持 PDF 简历上传，通过 MinerU 进行专业级 PDF 解析，提取结构化简历数据并生成四维能力标签。

**技术亮点**：
- 📄 **专业PDF解析**：集成 MinerU API，支持复杂排版简历的精准提取
- 🏷️ **四维能力标签自动生成**：
  - 行业标签（金融、电商、医疗等）
  - 技术标签（Python、SQL、机器学习等）
  - 产品标签（ToB、SaaS、0-1经验等）
  - 能力标签（需求分析、项目管理、数据分析等）

#### 模块三：多维匹配评估（五维度 + 精准评分）

**功能描述**：基于 JD 分析和简历解析结果，进行多维度深度匹配评估。

**评估模型**：

```
匹配度等级判定规则：
┌─────────────┬──────────────────────────────┐
│ 匹配等级     │ 判定条件                      │
├─────────────┼──────────────────────────────┤
│ 非常匹配     │ A3业务领域一致 + B1-B4全满足  │
│ 比较匹配     │ A3一致 + B1-B4满足3项        │
│ 匹配度还可以  │ A3相关 + B1-B4满足2项        │
│ 不是很匹配   │ A3不相关 或 B1-B4仅满足1项   │
│ 不匹配       │ B1-B4均不满足                │
└─────────────┴──────────────────────────────┘
```

**输出内容**：
- 📊 匹配度评分（0-100分）
- ✅ 五维度匹配状态（✅/⚠️/❌）
- 💪 优势亮点提取
- 🎯 差距分析与弥补建议
- 📝 面试重点建议

#### 模块四：公司智能分析（联网搜索 + AI场景挖掘）

**功能描述**：基于公司名称和岗位信息，联网搜索实时信息，生成公司画像和面试洞察。

**分析框架**：
- 🏢 **公司画像**：行业、发展阶段、规模、主要产品、商业模式
- 🤖 **AI场景分析**（特色功能）：
  - 当前AI应用场景
  - 潜在AI机会
  - 行业AI趋势
  - 该岗位的AI工作重点
- ⚔️ **竞品分析**：直接竞争对手、竞争地位、差异化点
- 💡 **面试洞察**：公司文化、关键挑战、成长机会、面试建议

#### 模块五：智能面试准备（PREP结构化 + 个性化生成）

**功能描述**：基于匹配分析和公司分析，生成个性化面试准备材料。

**核心产出**：

| 内容 | 描述 |
|------|------|
| **自我介绍** | 1分钟版（150-200字）+ 2分钟版（300-400字）|
| **项目推荐** | 1-2个最匹配项目 + STAR故事大纲 + 预测追问 |
| **面试题库** | 关于你/关于公司/关于未来 三大类问题 |
| **PREP回答** | Point-Reason-Example-Point 结构化回答 |
| **面试策略** | 目标印象 + 核心信息 + 避免话题 + 反问清单 |

#### 模块六：简历定向优化（关键词注入 + 差距弥补）

**功能描述**：基于匹配度差距，智能优化简历内容，提升与目标岗位的匹配度。

**优化策略**：
- 🔑 **关键词注入**：自然融入JD中的技术栈和能力关键词
- 🎯 **差距弥补**：针对匹配差距调整表述方式
- 💎 **亮点强化**：量化成果，使用STAR法则优化描述
- ✏️ **表述优化**：主动动词开头，结构清晰

**输出内容**：
- 原文 vs 优化后对比
- 每处修改的理由说明
- 对应的JD要求映射
- 预估匹配度提升

#### 模块七：面试教练（AI实时点评）

**功能描述**：用户输入面试问题和自己的回答，AI 教练给出专业点评和优化建议。

**点评维度**：
- 🚨 **必改项**：回答中的硬伤和错误
- 💡 **优化建议**：可以加强的地方
- ✨ **表达润色**：更专业的表达方式
- ⭐ **亮点识别**：做得好的地方
- 📈 **整体评分**：1-10分评估

#### 模块八：投递跟踪管理

**功能描述**：统一管理所有投递记录，跟踪投递状态，支持面试安排管理。

**核心功能**：
- 📋 投递记录自动关联岗位分析
- 🔄 状态流转管理（待投递→已投递→面试邀约→已面试→已通过/已拒绝）
- 📅 面试日程管理
- 📊 投递数据统计

### 3.2 核心技术创新

#### 创新一：多Agent DAG编排系统

**设计目标**：实现多个AI Agent的依赖管理和并行执行，提升处理效率。

**核心实现**：
```typescript
// DAG 执行器核心逻辑
class DAGExecutor {
  // 节点依赖管理
  private nodes: Map<string, DAGNodeConfig>;
  
  // 并行执行同层节点
  async execute(): Promise<DAGState> {
    while (hasExecutableNodes()) {
      // 找出所有依赖已满足的节点
      const executableNodes = findExecutableNodes();
      // 并行执行
      await Promise.all(executableNodes.map(executeNode));
    }
  }
}
```

**执行流程示例（JD解析）**：
```
Phase 1: JD预处理 (并行: 图片OCR / 文本清洗)
    │
    ▼
Phase 2: JD结构化
    │
    ▼
Phase 3: A维度分析 ──┬── B维度分析 (并行执行)
                    │
                    ▼
Phase 4: 完成，汇总结果
```

#### 创新二：双通道LLM架构

**设计目的**：
- 成本优化：不同任务使用不同级别模型
- 可用性保障：主通道故障可切换备用
- 灵活扩展：便于接入新模型

**模型配置策略**：

| Agent | 模型选择 | 选型理由 |
|-------|---------|---------|

| JD预处理(图片) | qwen-vl-max | 视觉理解能力 |
| JD结构化 | qwen-plus | 结构化提取，均衡型 |
| A/B维度分析 | qwen-max | 深度分析，最强推理 |

| 公司分析 | qwen-plus + 联网搜索 | 需要实时信息 |
